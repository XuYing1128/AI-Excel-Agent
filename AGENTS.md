# AI-Excel-Agent Agent Rules

本文件是项目级强约束。Codex 在本项目内处理任何 Excel / CSV / 表格任务时，必须自动遵守，无需用户重复说明。
处理表格任务时必须使用 `skills/xlsx_pro/SKILL.md` 中的 `xlsx_pro` 技能。

## 项目定位

AI-Excel-Agent 是一个本地 Excel / CSV 表格自动化工作台，用于生成、修改、清洗、分析和校验表格。项目长期使用，默认保护用户原始文件，默认把生成结果放入 `outputs/`。

V1 同时提供 Windows 本地 Streamlit 页面。页面只监听 `127.0.0.1`，不提供账号、云端、多租户或队列能力。

## V1 网页强制边界

- 大模型永远不能直接读取并修改具体单元格。模型只允许做意图理解、一次性澄清、结果解释和建议性审查。
- 模板选择、输入文件复制、数据清洗、公式、样式、图表、生成和客观校验必须由确定性 Python 代码完成。
- 用户未确认结构化 `TaskSpec` 前，不允许调用生成内核。
- 主观模型审查只能输出建议 JSON；失败、超时或未启用不得阻止 Excel 下载。
- 自定义 API 只允许接收用户需求、文件名、TaskSpec、校验摘要、工作簿结构摘要和运行摘要；不得发送完整工作簿数据。
- 自定义 API 配置必须保存在本机忽略目录，不能进入 Git；API 失败时必须回退到本地规则。
- 完整性、重复、公式错误、时间冲突、字段异常和数据缺失不消耗模型 token，必须使用确定性校验。
- V1 不做真实 Excel 重算，不依赖 LibreOffice、Excel COM 或 OfficeCLI。

## 默认工作流

每次接到表格任务时必须按顺序执行：

1. 解析用户需求，识别目标、输入、输出、计算逻辑和风险点。
2. 判断表格类型：预算、报价、库存、销售、电商、项目、排班课程、考勤、财务模型、Dashboard 或通用表格。
3. 检查是否有输入文件；没有输入文件时按合理默认样例生成，并说明默认假设。
4. 生成简短计划，说明将创建或修改哪些 workbook、sheet、公式区和校验方式。
5. 选择已有模板；没有合适模板时创建新模板或扩展 `templates/`。
6. 用 pandas 读取、清洗、合并、聚合数据；保留中文列名。
7. 用 openpyxl / xlsxwriter 生成或修改 workbook、公式、样式、图表和数据验证。
8. 所有计算全部写入 Excel 公式，不把计算结果硬编码为静态值。
9. 设置标题、表头、输入区、计算区、列宽、冻结窗格、筛选、数字格式、打印设置和必要图表。
10. 网页任务保存到 `outputs/tasks/<task_id>/`；修改已有文件前先复制到任务的 `input/` 或 `backups/`。
11. 运行校验器，检查文件可打开、sheet、表头、错误值、公式引用、公式覆盖、列宽、说明页和关键列。
12. 如有错误，必须自动修复后再次校验；不能静默忽略。
13. 最终说明输出文件路径、主要 sheet、公式区域、校验结果、默认假设和人工复核点。

## 强制规则

- 不允许把 `SUM`、`AVERAGE`、增长率、占比、毛利率、ROI、出勤率、库存周转等计算结果硬编码进 Excel；必须写公式。
- 不允许覆盖用户原文件；修改前必须复制到 `outputs/` 或 `backups/`。
- 不允许静默忽略错误、异常、公式风险或校验失败。
- 不允许交付无法打开的 `.xlsx`。
- 不允许把敏感数据上传到外部服务，除非用户明确同意。
- 财务、税务、工资、审计、绩效、奖金、合同报价、投融资等高风险表格必须标明“需要人工复核”。
- pandas 负责数据读取、清洗、合并和聚合；openpyxl / xlsxwriter 负责 workbook、公式、样式、图表、冻结窗格、筛选和打印设置。
- 如果本机没有 Excel 或 LibreOffice，不得声称已完成真实公式重算，只能说明已写入公式并完成静态校验。

## 文件规范

- 输入文件：`examples/input/` 或 `inputs/`
- 输出文件：`outputs/tasks/<task_id>/output/result.xlsx`
- 校验报告：`outputs/tasks/<task_id>/reports/validation.json`
- 任务规范：`outputs/tasks/<task_id>/task_spec.json`
- 运行日志：`outputs/tasks/<task_id>/run_log.json`
- 历史索引：`outputs/manifest.json`
- 临时文件：`temp/`
- 备份文件：`backups/`
- 模板文件：`templates/`
- 技能说明：`skills/xlsx_pro/`
- 业务结果不得散落在项目根目录。
- 旧版根目录输出只能移动到 `outputs/legacy/`，不得删除用户文件。

## 表格质量标准

- 每个 workbook 至少有一个说明页、README sheet 或清晰备注区。
- 所有重要表头必须清楚、稳定、不可为空。
- 数字、金额、百分比、日期格式必须正确。
- 列宽必须自适应或手动合理设置，不能明显截断关键文字。
- 数据区域必须能筛选，表头应冻结。
- 关键公式必须覆盖完整数据区，不能中断或漏填。
- Dashboard 必须把数据源 sheet 和展示 sheet 分离。
- 涉及图表时，图表引用范围必须清楚且跟随数据区。
- 高风险结果必须给人工复核提醒。

## 失败处理

- 如果公式重算失败，必须说明原因；不得假装已经重算。
- 如果无法渲染预览，继续执行结构校验并说明预览限制。
- 如果输入数据混乱，先生成 `clean_report`，不得直接猜测字段口径。
- 如果需求不清楚但可以合理默认，先按默认方案生成，并在结果中说明默认假设。
- 如果校验失败，先修复再重新校验；确实无法修复时，说明失败项、影响范围和下一步需要的用户信息。
