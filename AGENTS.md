# AI-Excel-Agent Agent Rules

本文件是项目级强约束。Codex 在本项目内处理任何 Excel / CSV / 表格任务时，必须自动遵守，无需用户重复说明。
处理表格任务时必须使用 `skills/xlsx_pro/SKILL.md` 中的 `xlsx_pro` 技能。

## 项目定位

AI-Excel-Agent 是一个本地 Excel / CSV 表格自动化工作台，用于生成、修改、清洗、分析和校验表格。项目长期使用，默认保护用户原始文件，默认把生成结果放入 `outputs/`。

V1 同时提供 Windows 本地 Streamlit 页面。页面只监听 `127.0.0.1`，不提供账号、云端、多租户或队列能力。

## 网页与大模型强制边界

- 用户启用“大模型生成”后，大模型必须通过工具调用提交结构化 workbook blueprint，不能只返回解释、Markdown 或建议。
- 大模型可以设计列、两级表头、公式模板、排序、小计、总计、条件格式和图表，但不能直接打开文件、任意执行代码或直接写改单元格。
- blueprint 必须由本地受控 Python 工具执行；文件复制、数据清洗、单元格写入、公式渲染、样式、图表、生成和客观校验仍由确定性代码完成。
- 大模型首次方案缺项时，必须把本地结构检查和需求一致性检查结果返回给模型继续修订；未满足硬性要求不得调用完成工具。
- 用户已确认的图表、说明区、汇总、文件名、模板保留等选项是硬性要求，主观审查不得反向建议删除。
- 用户未确认结构化 `TaskSpec` 前，不允许调用生成内核。
- `TaskSpec` 必须尽可能保存用户要求的标题、列、数据来源、公式语义、汇总方式和预期数据行数。用户已明确结构时，禁止退化为无关通用模板。
- 主观模型审查只能输出建议 JSON；失败、超时或未启用不得阻止 Excel 下载。
- 自定义 API 默认只接收用户需求、文件名、TaskSpec、校验摘要、工作簿结构摘要和运行摘要；完整上传数据只有在用户明确同意时才可发送。
- 自定义 API 配置必须保存在本机忽略目录，不能进入 Git；意图理解或主观审查失败可回退本地规则，大模型生成失败时必须明确报错，不能静默改用无关模板。
- 完整性、重复、公式错误、时间冲突、字段异常和数据缺失不消耗模型 token，必须使用确定性校验。
- 主流程不强制依赖 LibreOffice、Excel COM 或 OfficeCLI；检测到可用重算引擎时可以重算，否则必须明确标记为仅静态校验。
- 网页中的接口配置必须放在独立页面，并通过显式保存才生效，不能与表格制作表单混排。
- 生成结果必须支持页面内预览、报告预览和继续修改；每次修改生成新版本，不覆盖前一版。

## 默认工作流

每次接到表格任务时必须按顺序执行：

1. 解析用户需求，识别目标、输入、输出、计算逻辑和风险点。
2. 判断表格类型：预算、报价、库存、销售、电商、项目、排班课程、考勤、财务模型、Dashboard 或通用表格。
3. 检查需求完整度：数据来源、字段及顺序、计算口径、分组排序、小计总计、图表、时间范围和金额口径；只询问会改变结果的缺失项，并给出可直接照填的示例。
4. 用户补充后必须重新构建 `TaskSpec` 和内容方案，不得继续使用补充前的旧判断。
5. 检查是否有输入文件；没有输入文件时按合理默认样例生成，并说明默认假设。
6. 生成简短计划，说明将创建或修改哪些 workbook、sheet、公式区和校验方式。
7. 启用大模型生成时，由模型调用本地 `build_workbook` 工具提交 blueprint；未启用时再选择匹配模板或本地生成器。
8. 用 pandas 读取、清洗、合并、聚合数据；保留中文列名。
9. 用 openpyxl / xlsxwriter 生成或修改 workbook、公式、样式、图表和数据验证。
10. 所有计算全部写入 Excel 公式，不把计算结果硬编码为静态值。
11. 设置标题、表头、输入区、计算区、列宽、冻结窗格、筛选、数字格式、打印设置和必要图表。
12. 网页任务保存到 `outputs/tasks/<task_id>/`；修改已有文件前先复制到任务的 `input/` 或 `backups/`。
13. 运行校验器，检查文件可打开、sheet、表头、错误值、公式引用、公式覆盖、列宽、说明页和关键列。
14. 对照 `TaskSpec` 进行需求一致性检查：标题、要求列、数据条数、公式列和图表要求不得偏离。
15. 如有错误或需求缺项，必须自动修复或把精确反馈交给模型修订后再次校验；不能静默忽略。
16. 页面提供表格、生成方案、检查报告和审查建议预览，并允许把审查建议直接带入下一次修改。
17. 最终说明输出文件路径、主要 sheet、公式区域、校验结果、默认假设和人工复核点。

## 强制规则

- 不允许把 `SUM`、`AVERAGE`、增长率、占比、毛利率、ROI、出勤率、库存周转等计算结果硬编码进 Excel；必须写公式。
- 不允许覆盖用户原文件；修改前必须复制到 `outputs/` 或 `backups/`。
- 不允许静默忽略错误、异常、公式风险或校验失败。
- 不允许交付无法打开的 `.xlsx`。
- 不允许在用户已明确列、数据或计算规则时仍交付无关标准模板。
- 用户启用大模型生成后，不允许因模型失败而静默回退到不相干模板；必须明确失败原因并保留可重试状态。
- 不允许仅凭模型声称“已完成”就交付；必须以本地生成结果和确定性校验为准。
- 数据文件和模板文件必须分开处理；模板中的示例数据默认不得作为业务数据。
- 模板模式必须由用户确认：`reference` 仅参考视觉，`flexible` 需求优先且允许增删，`strict` 保持字段顺序和工作表结构，冲突时必须停止并提示。
- `.xls` 作为数据时使用兼容读取器；作为模板时先安全转换为 `.xlsx`，不得假装已完整保留旧格式。
- 不允许把一次生成视为任务终点；用户提出修改或采用审查建议时，必须生成保留旧版的新版本。
- 不允许把敏感数据上传到外部服务，除非用户明确同意。
- 财务、税务、工资、审计、绩效、奖金、合同报价、投融资等高风险表格必须标明“需要人工复核”。
- pandas 负责数据读取、清洗、合并和聚合；openpyxl / xlsxwriter 负责 workbook、公式、样式、图表、冻结窗格、筛选和打印设置。
- 如果本机没有 Excel 或 LibreOffice，不得声称已完成真实公式重算，只能说明已写入公式并完成静态校验。

## 文件规范

- 输入文件：`examples/input/` 或 `inputs/`
- 输出文件：`outputs/tasks/<task_id>/output/<根据内容生成或用户指定的名称>.xlsx`
- 校验报告：`outputs/tasks/<task_id>/reports/validation.json`
- 任务规范：`outputs/tasks/<task_id>/task_spec.json`
- 运行日志：`outputs/tasks/<task_id>/run_log.json`
- 历史索引：`outputs/manifest.json`
- 临时文件：`temp/`
- 备份文件：`backups/`
- 模板文件：`templates/`
- 技能说明：`skills/xlsx_pro/`
- 业务结果不得散落在项目根目录。
- 输出文件名必须根据内容生成或由用户明确填写，不能所有任务都叫 `result.xlsx` 或“生成结果.xlsx”。
- 旧版根目录输出只能移动到 `outputs/legacy/`，不得删除用户文件。

## 表格质量标准

- 每个 workbook 至少有一个说明页、README sheet 或清晰备注区。
- 所有重要表头必须清楚、稳定、不可为空。
- 数字、金额、百分比、日期格式必须正确。
- 列宽必须自适应或手动合理设置，不能明显截断关键文字。
- 数据区域必须能筛选，表头应冻结。
- 关键公式必须覆盖完整数据区，不能中断或漏填。
- 生成结果的标题、表头、数据条数和计算列必须与确认方案一致。
- Dashboard 必须把数据源 sheet 和展示 sheet 分离。
- 涉及图表时，图表引用范围必须清楚且跟随数据区。
- 图表必须使用跨 Excel/WPS/网页 Office 可读取的稳定数据源；不得只引用未重算的隐藏公式而产生空图。
- 高风险结果必须给人工复核提醒。

## 失败处理

- 如果公式重算失败，必须说明原因；不得假装已经重算。
- 如果无法渲染预览，继续执行结构校验并说明预览限制。
- 如果输入数据混乱，先生成 `clean_report`，不得直接猜测字段口径。
- 如果需求不清楚但可以合理默认，先按默认方案生成，并在结果中说明默认假设。
- 如果需求中已给出结构化数据或明确列名，优先直接解析并生成，不得反复追问已提供的信息。
- 如果校验失败，先修复再重新校验；确实无法修复时，说明失败项、影响范围和下一步需要的用户信息。
