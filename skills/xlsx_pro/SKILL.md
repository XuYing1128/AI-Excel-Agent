# xlsx_pro

本技能是 AI-Excel-Agent 项目处理 Excel / XLSX / CSV / 表格任务的专用执行规范。触发后必须同时遵守项目根目录 `AGENTS.md`。

## 触发场景

使用本技能处理以下任务：

- 创建、修改、修复、格式化或校验 `.xlsx`、`.xlsm`、`.csv`、`.tsv` 表格。
- 生成预算表、报价单、库存表、销售报表、电商分析、项目计划、排班/课程表、考勤表、财务模型、Dashboard。
- 清洗原始数据、合并多表、生成汇总、透视风格报表、图表或 KPI 看板。
- 检查公式错误、表头缺失、列宽、筛选、冻结窗格、打印设置、说明页和公式覆盖范围。

## 不触发场景

不要为以下任务使用本技能：

- 纯文本写作、代码重构、网页开发、图片生成、PPT/DOCX 制作，且不涉及表格文件。
- 只需要口头解释 Excel 函数而不需要创建、修改或校验文件。
- 用户明确要求不使用本项目工具链。

## 标准工作流

1. 解析需求：确认目标、输入文件、输出文件、计算逻辑、风险点和默认假设。
2. 判断表格类型：预算、报价、库存、销售、电商、项目、排班课程、考勤、财务模型、Dashboard 或通用表格。
3. 检查输入：输入文件优先位于 `examples/input/` 或 `inputs/`；没有输入时使用合理样例并说明。
4. 制定简短计划：列出 workbook、主要 sheet、公式区、图表和校验方式。
5. 网页任务必须先生成结构化 `TaskSpec`，用普通语言展示并等待用户确认；未确认不得生成。
6. 用户明确给出标题、列、数据和计算规则时，先生成内容方案并逐项对照；禁止回退到无关通用模板。
7. 选择模板：只有需求与模板结构确实匹配时才使用 `templates/`；否则按内容方案生成。
8. 处理数据：用 pandas 读取、清洗、合并、聚合、统计，保留中文列名。
9. 生成表格：用 openpyxl / xlsxwriter 写入 sheet、公式、样式、筛选、冻结窗格、图表、数据验证和打印设置。
10. 写公式：所有派生结果默认写 Excel 公式，不硬编码计算值。
11. 保存文件：网页任务输出到 `outputs/tasks/<task_id>/`，文件名根据内容生成或使用用户指定名称；修改已有文件前复制到任务 `input/` 或 `backups/`。
12. 校验文件：运行确定性校验器并保存 `reports/validation.json`，同时核对标题、要求列、数据行和公式列是否符合确认方案。
13. 修复再验：若校验失败或发现错误，自动修复并再次校验。
14. 页面预览：提供表格、生成方案、检查报告和审查建议预览。
15. 持续修改：审查建议应可直接转为下一次修改要求；修改生成新版本，不覆盖原版本。
16. 交付说明：说明文件路径、sheet、关键公式、重算状态、校验结果和人工复核点。

## V1 模型边界

- 模型只做意图理解、最多一轮澄清、结果解释和建议性审查。
- 模型不能生成、修改或覆盖具体单元格，不能替代 pandas/openpyxl/xlsxwriter。
- 客观正确性全部由确定性代码检查，不把完整敏感数据默认发送给外部模型。
- 主观审查失败或未启用不影响下载；两个模型都 pass 也不能证明客观正确。
- 自定义 API 只能辅助理解需求和生成主观建议，不能读取完整工作簿数据，不能修改单元格。
- 自定义 API 可以整理用户文本中明确给出的标题、列和计算语义，但不能输出单元格地址或任意 Excel 公式。
- API 连接失败时回退到本地规则和确定性校验，不能阻塞表格生成与下载。
- 接口设置必须位于独立页面，并通过显式保存才生效，避免误触修改。

## 工具选择规则

- pandas：读取 CSV/XLSX、识别表头、清洗数据、合并多源、去重、聚合、统计、生成 clean_report 所需数据。
- openpyxl：修改已有 xlsx、保留工作簿结构、写公式、样式、数据验证、冻结窗格、筛选、列宽、页眉页脚、打印设置。
- xlsxwriter：新建复杂报表、复杂图表、Dashboard、图表密集型 workbook。
- LibreOffice headless：检测到 `soffice` / `libreoffice` 时，用于打开保存并重算公式。
- Excel COM：Windows + Microsoft Office + `win32com.client` 可用时，用于真实 Excel 重算。
- OfficeCLI：可用时用于渲染 HTML/PNG 预览；不可用时不阻塞主流程，继续结构校验。

## 公式规则

- 默认写 Excel 公式，不硬编码 `SUM`、`AVERAGE`、`COUNTIF(S)`、`SUMIF(S)`、`XLOOKUP`、增长率、占比、毛利率、ROI、库存周转、客单价、GMV、出勤率等结果。
- 所有公式必须考虑除零、空值和错误值；优先使用 `IFERROR`，空模板行应返回空字符串或 0。
- 公式范围必须随实际数据行数自动调整，不能漏行、断行或引用无关区域。
- 跨 sheet 公式必须使用清晰 sheet 名和稳定引用范围。
- 关键假设必须放在可见输入区，不要隐藏在公式常量里。
- 生成后必须检查 `#REF!`、`#DIV/0!`、`#VALUE!`、`#NAME?`、`#N/A` 等错误。

## 样式规则

- 每个 workbook 至少包含 `Instructions`、`README` 或清晰说明区。
- 标题、表头、输入区、计算区、汇总区应有明确样式区分。
- 日期、金额、数字、百分比、负数红字等格式必须正确。
- 数据区域必须启用筛选，表头应冻结。
- 列宽应自适应或手动合理设置，不能明显截断关键内容。
- 设置必要的打印方向、边距、页眉页脚和打印区域。
- 不在可筛选数据区内部使用破坏筛选的合并单元格。

## 数据清洗规则

- 自动识别表头，保留中文列名，不随意翻译业务字段。
- 去除空行空列、重复行，标准化日期、金额、数量和百分比字段。
- 生成缺失值报告、异常值报告和清洗动作记录。
- 输入数据混乱或字段口径不明时，先生成 `clean_report`，不得直接猜测业务含义。
- 原始输入只读处理；需要修改时另存到 `outputs/` 或先备份到 `backups/`。

## 图表和仪表盘规则

- 报表和 Dashboard 应优先使用真实 Excel 图表，而不是截图或静态图片。
- Dashboard 必须分离数据源 sheet 和展示 sheet，例如 `SourceData` + `Dashboard`。
- 至少包含适合任务的 KPI、Top N、趋势、分类占比或明细表。
- 图表引用范围必须清楚，随数据行数或汇总区域同步更新。
- 图表标题、轴标签、数字格式和布局必须可读。
- 网页应提供工作表内容预览；预览不可用时仍需保留结构校验和下载。

## 校验规则

生成或修改后必须校验：

- 文件能否用 openpyxl 打开。
- sheet 名称是否合法，是否存在空 sheet。
- 是否存在说明页或备注区。
- 是否存在 Excel 错误值或公式错误标记。
- 是否存在明显越界公式引用。
- 是否存在合并单元格破坏数据区筛选。
- 是否隐藏了关键 sheet。
- 表头是否完整，关键列是否为空。
- 公式区域和数据区域长度是否一致。
- 列宽、冻结窗格、筛选和打印设置是否合理。
- 标题、用户要求的列、识别到的数据条数、公式列和图表要求是否与确认方案一致。

## 重算规则

- 优先尝试 LibreOffice headless 重算。
- Windows + Office 可用时，可尝试 Excel COM 真实重算。
- 两者都不可用时，必须说明“已写入公式但未完成真实 Excel 引擎重算”，并继续完成静态校验。
- 重算失败不得静默忽略，不得声称已完成重算。

## 失败恢复

- 文件打不开：检查路径、扩展名、写入权限，重新保存或生成新文件。
- 公式错误：定位具体 sheet、单元格、引用范围，局部修复后再次校验。
- 图表损坏：保留数据和公式，简化图表引用范围后重新生成。
- 预览失败：说明 OfficeCLI 或渲染工具不可用，继续结构校验。
- 输入混乱：生成 `clean_report`，列出缺失、重复、异常和无法识别字段。
- 需求不清楚：能合理默认就先生成，并在交付说明中列出默认假设；不能合理默认时再问用户。
- 审查发现偏差：把问题和建议整理成可编辑的下一次修改要求，生成新版本并重新校验。

## 高风险场景提醒

财务、税务、工资、审计、绩效、奖金、合同报价、投融资、库存盘点和法务相关表格必须标明“需要人工复核”。AI 只能生成结构、公式、清洗报告和校验报告，不能替代专业审核或审批。

## 输出要求

每次完成表格任务后，必须告诉用户：

- 生成或修改了哪个文件。
- 包含哪些主要 sheet。
- 使用了哪些关键公式或公式区域。
- 是否完成真实重算；若未完成，说明原因。
- 校验是否通过；若有 warning/error，说明处理结果。
- 哪些地方需要人工复核。

## 中文示例任务

1. “帮我生成一个个人月度收支预算表，包含预算、实际、差异和分类汇总。”
2. “根据这个销售 CSV 生成销售月报，按品类和区域汇总，并加趋势图。”
3. “把库存表清洗后生成进销存表，自动提示低于安全库存的 SKU。”
4. “做一个产品报价单模板，数量、单价、折扣、税率、未税金额和含税总价都用公式。”
5. “根据电商订单数据分析 GMV、退款率、毛利和渠道占比。”
6. “生成项目计划表，包含负责人、开始结束日期、工期、完成率和延期状态。”
7. “做一个员工排班表，按日期、班次、负责人和时长统计。”
8. “清洗这个考勤表，输出迟到、缺勤和出勤率统计。”
9. “生成一个简单利润测算模型，按月份计算收入、成本、净利润、毛利率和 ROI。”
10. “做一个综合 Dashboard，数据源和展示页分离，包含 KPI、Top N、趋势和分类占比。”
